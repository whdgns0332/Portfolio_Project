				(재고 확보를 위해 추가로 만드는 경우도 있다고 하고, 상황에 따라 목표생산량보다 더 적게 생산할 수도 있다고 함)
				(목표생산량의 설정은 그동안의 불량률 통계를 고려하여 실제 목표보다 추가해 이뤄지는데
					운이 좋다면 목표생산량보다 더 적게 생산하고도 필요한 정상품의 수를 맞출수도 있다)

			따라서 작업의 종료 기준은 목표생산량만큼 생산했느냐가 아니라 작업자의 판단 하에 결정되어야함
			(작업자의 정상적인 지시로 작업이 종료되었느냐, 설비고장이나 재료부족 등으로 작업이 비정상적 종료되었느냐를 구분해야함
				온도값이 오랫동안 범위를 벗어나는 등의 이상이 발생한다고 작업이 자동 중단되는 시스템의 존재를 고려하지는 않는다 - 즉 없다고 가정한다
				따라서 비정상적 상황이 발생시에 생산을 중단하는 것도 사용자인 만큼 더더욱 비정상적 종료를 확인해야한다
				이렇게 확인해 결과 테이블에 비정상 종료 여부를 기록하게 될 경우 생산이 필요한 작업지시의 목록을 찾는것이 더 수월해질것)]

			List로 진행할 작업지시 목록을 관리할 필요가 사라져서 이쪽 관련 구조가 더 간단해짐
			[결과 테이블에 기록된 생산량이 목표생산량을 넘는지를 확인하는 것보다 단지 비정상 종료인지를 확인하는 것이 훨씬 쉽다]

			{기기를 작동시키고 종료시키는 코드의 작성이 추후 요구됨}

별도의 프로젝트를 만들어 다음 기능을 가진 프로그램을 작성할것
	1. 좌측 상단에는 현재 시간을 표시할것
	2. 상단에는 ini파일로부터 받아온 PLC의 IP, Port를 출력할것
		(PLC에 연결할 때 실제로 해당 IP, Port를 사용한다)
	3. IP, Port Label 아래에는 현재 연결상태(Connected, Disconnected 등)를 표시하는 Label을 만들것
	4. 우측 상단에는 PLC, Auto 버튼을 생성할것
	5. PLC, Auto 버튼 아래에 시작, 종료 버튼을 생성할것
		시작 버튼을 누르면 다음과 같이 생산실적, 내경, 외경온도 데이터를 받아 DB에 업로드한다
			PLC - ini파일에 저장된 IP, Port값을 이용해 PLC에 연결해 데이터를 받아온다
			Auto - 데이터를 랜덤하게 생성한다
			데이터를 받거나 생성하는 주기는 1초로 한다 (실무에서는 1~2초, 최악의 경우에도 3초 내로 해야한다고 함)
		종료 버튼을 누르면 데이터 생성 및 DB 업로드를 중단한다
	6. 화면 중간~하단에는 받아온 데이터를 표시할 DataGridView를 생성한다
		(대량의 데이터를 화면에 표시할 수 있도록 글씨체의 크기를 크게 하지 말것
			또한 마우스의 사용이 가능하므로 스크롤이 가능하도록 버튼을 따로 만들 필요는 없음)

	7. 본래 프로그램에서 작업을 개시했을 때 DB에 FLAG Column을 통해 어떤 작업지시가 새로 수행되는가를 나타내고
					(Flag Column을 추가할 필요성이 있는듯)
	    현 프로그램에서는 새로 수행되는 작업지시가 파악되면 해당 작업에 대한 생산품 수 값을 확인해 받아온다
		작업이 종료되면 현 프로그램에서는 PLC에 저장된 생산품 수의 값을 0으로 초기화시킨다

	그 외에 본래 프로그램에 다음과 같은 추가사항을 구현한다
		차트의 데이터는 분 단위로 표시한다
		(DB에 저장되는 데이터는 초당 1회, 즉 분당 60개의 데이터가 저장되나 실제 사용할 것은 60개중 1개 뿐이므로 60개당 1개를 뽑아낼 방법이 요구됨)
		(1시간 3600개의 데이터를 모두 받아온 다음 for(int i = 0; i < 3600; i += 60)같은 방법을 사용해도 될 듯하고 - 이 방법은 받아올 때마다 데이터의 초 단위가 일치한다는 보장이 없어 좋다고는 못함
			데이터를 받아오는 조건에 시간의 초 부분이 00인 데이터만 받아오는 방법도 가능할 것이다
			아니면 테이블에 별도의 bit 타입 Column을 생성한 뒤 IO서버 프로그램에서 업로드할 데이터의 초 단위가 00인 경우에만 해당 Column의 값을 True로 해 업로드하는 것으로
				간단하게 처리가 가능해 보인다)

Config 테이블
QMCode, PLC에서 해당 값이 저장될 메모리 위치, Status값(Read/Write)

본래 프로그램에서 작업을 개시하면, Config 테이블의 PQ 코드에 대한 Status 값을 Read로 바꾼다 (TP1, TP2는 항상 Read로 유지)
	작업을 종료하면, PQ코드에 대한 Status값을 Write로 바꾼다
IO서버 프로그램에서 작업 진행시, 매초마다 Config 테이블에서 각 코드에 대한 Status값을 읽는다
	Read라면 PLC로부터 데이터를 읽어와 Q_Control 테이블에 저장하며
	Write라면 PLC의 해당 코드에 대한 메모리 영역의 값을 0으로 초기화한다
